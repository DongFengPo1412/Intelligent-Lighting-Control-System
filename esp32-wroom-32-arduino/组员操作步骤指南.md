## 🛠️ 第一步：物理硬件接线与电平对齐

由于 ESP32 目前使用的是 **USB 连电脑供电**，且其 **`VIN` 引脚已经接入灯板的 `V+` 给灯板供电**。为了防止双路 5V 电源发生电流倒灌与冲突，**千万不要连接 Jetson Nano 的 5V 供电线到 ESP32**。

你只需要用杜邦线连接以下 **3 根线** 即可完成双机异步串口通信：

| Jetson Nano J41 40-Pin 排针引脚 | ➔ | ESP32-WROOM-32 对应引脚 | 引脚功能与说明 |
| :--- | :---: | :--- | :--- |
| **Pin 8 (UART1_TXD)** | ➔ | **GPIO 25** (作为 RX2) | Jetson 发送 ➔ ESP32 接收 |
| **Pin 10 (UART1_RXD)**| ➔ | **GPIO 26** (作为 TX2) | ESP32 发送 ➔ Jetson 接收 |
| **Pin 9 (GND)** | ➔ | **GND** | **共用接地参考线** (必须连接，以对齐基准电平) |

> [!WARNING]
> **共地（GND 互连）是串口通信的前提**。如果双方没有接在同一个地线上，物理参考电平会发生漂移，导致串口传输严重乱码，甚至由于瞬时电势差击穿引脚。
> ESP32 固件中已对 `GPIO 25` (RX2) 和 `GPIO 26` (TX2) 启用了物理 `INPUT_PULLUP`（上拉电阻），即使杜邦线意外松动，引脚也不会因为悬空高阻态吸收杂波而误触发切模。

## ⚙️ 第二步：Jetson Nano 系统级串口防占配置

Jetson Nano 的默认操作系统（Ubuntu 18.04 LTS）开机时会自动在串口 `/dev/ttyTHS1` 上启动 `nvgetty` 终端监听服务。如果这个后台服务不关闭，它会不断往串口写入调试日志，并与你的 Python 脚本争抢数据接收权，导致数据乱码或丢失。

请在 **Jetson Nano 的终端** 执行以下命令，彻底释放串口：

```bash
# 1. 停止当前串口终端服务
sudo systemctl stop nvgetty

# 2. 禁用 nvgetty 服务开机自启
sudo systemctl disable nvgetty

# 3. 将当前用户加入 dialout（串口组）和 video（摄像头组），实现免 sudo 运行
sudo usermod -aG dialout $USER
sudo usermod -aG video $USER

# 4. 重启系统以应用用户组更改
sudo reboot
```

## 🚀 第四步：AI 视觉交互功能设计与指令映射

灯光板支持的指令集分为**模式切换指令（数字）**和**实时音律随动流（字节流）**。Jetson Nano 通过物理串口，以 `115200` 波波特率向 ESP32 发送指令。

### 1. 灯光切换指令对照表（0 ~ 22 号模式）
Jetson Nano 发送裸数字（如 `11\n`），ESP32 会自动切模并运行对应的物理特效：

| 指令 (ASCII 字符) | ESP32 运行效果 | 适合 AI 联动场景 |
| :---: | :--- | :--- |
| **`0`** | 待机 / 关闭矩阵 | 画面中完全无人、手势“攥拳/拳头” |
| **`4`** | 幻彩霓虹 | 基础手势 1（单指） |
| **`5`** | 呼吸极光 | 基础手势 2（剪刀手） |
| **`6`** | 流星划过 | 基础手势 3（OK 手势） |
| **`10`**| 中心波纹 | 画面中有人、但无特定动作（默认待命状态） |
| **`11`**| 笑脸盈盈 (Emoji 1) | **人脸识别**：检测到人脸，且表情为“微笑/开心” |
| **`12`**| 垂头丧气 (Emoji 2) | **人脸识别**：检测到人脸，且表情为“难过/沮丧” |
| **`13`**| 面无表情 (Emoji 3) | **人脸识别**：检测到人脸，且表情为“平静/严肃” |
| **`14`**| 瞬息万变 | 画面中同时出现多张人脸或多人活动 |
| **`19`**| 启动贪吃蛇游戏 | 手势“比划 4” ➔ 瞬间开启贪吃蛇 |
| **`21`**| 启动俄罗斯方块游戏 | 手势“比划 5” ➔ 瞬间开启俄罗斯方块 |

## 📝 第五步：Jetson 侧 Python 联动核心代码模板

为了方便还没拿到最终代码的同学进行开发和对接，这里提供一份**带防高频阻塞限流、基于 MediaPipe Hands 的手势识别与人脸联动 Python 核心框架代码**。

你可以直接在 Jetson Nano 上创建一个名为 `jetson_ai_bridge.py` 的文件，并将此代码贴入运行。

```python
# -*- coding: utf-8 -*-
"""
Intelligent Lighting System - Jetson Nano AI Bridge
NVIDIA Jetson Nano ➔ ESP32-WROOM-32 串口联动节点
功能：人脸与手势双模态识别，通过串口发送对应的模式指令到 ESP32 灯阵
"""

import sys
import time
import serial
import cv2
import numpy as np
import mediapipe as mp

# ==================== 1. 配置与初始化串口 ====================
PORT = "/dev/ttyTHS1"  # Jetson Nano UART1 物理设备名
BAUDRATE = 115200

try:
    ser = serial.Serial(
        port=PORT,
        baudrate=BAUDRATE,
        bytesize=serial.EIGHTBITS,
        parity=serial.PARITY_NONE,
        stopbits=serial.STOPBITS_ONE,
        timeout=0.1
    )
    print(f"[OK] 串口 {PORT} 成功打开，波特率: {BAUDRATE}")
except Exception as e:
    print(f"[ERROR] 无法打开串口 {PORT}。请检查接线，或运行: sudo chmod 666 {PORT}")
    print(f"报错详情: {e}")
    sys.exit(1)

# ==================== 2. 限流与流速控制器 ====================
class ESP32Controller:
    def __init__(self, serial_conn):
        self.ser = serial_conn
        self.last_sent_mode = -1
        self.last_sent_time = 0.0
        self.cooldown = 0.6  # 发送冷却时间（秒），防止 AI 帧率过高导致 ESP32 串口缓冲区爆满

    def send_mode(self, mode_num):
        current_time = time.time()
        # 1. 过滤连续重复的模式
        if mode_num == self.last_sent_mode:
            return
        # 2. 限制发送频率，防止高频震荡
        if current_time - self.last_sent_time < self.cooldown:
            return

        try:
            # 必须追加 '\n'，ESP32 固件依赖换行符进行快速断帧解析
            cmd = f"{mode_num}\n"
            self.ser.write(cmd.encode("ascii"))
            self.ser.flush()  # 荡平本地缓冲区，确保数据即时发到物理线路
            self.last_sent_mode = mode_num
            self.last_sent_time = current_time
            print(f"[TX ➔] 成功下发模式指令: {mode_num}")
        except Exception as ex:
            print(f"[TX ❌] 写入串口时异常: {ex}")

controller = ESP32Controller(ser)

# ==================== 3. 初始化 AI 视觉检测器 ====================
# 初始化 MediaPipe Hands
mp_hands = mp.solutions.hands
hands = mp_hands.Hands(
    static_image_mode=False,
    max_num_hands=1,
    min_detection_confidence=0.7,
    min_tracking_confidence=0.5
)
mp_draw = mp.solutions.drawing_utils

# 初始化 OpenCV 自带的轻量级人脸检测器 (若你有更高级的人脸/表情分类器可在此替换)
face_cascade = cv2.CascadeClassifier(cv2.data.haarcascades + 'haarcascade_frontalface_default.xml')

# ==================== 4. 主事件循环与决策状态机 ====================
cap = cv2.VideoCapture(0)  # 打开默认摄像头 (通常为 /dev/video0)
if not cap.isOpened():
    print("[ERROR] 无法启动 USB 摄像头。请检查连接或使用 ls /dev/video* 查看设备节点。")
    sys.exit(1)

print("\n>>> AI 联动核心程序已就绪。将画面置于前台，按 'q' 键退出程序。")

while True:
    ret, frame = cap.read()
    if not ret:
        print("[WARN] 无法获取摄像头帧，尝试跳过...")
        time.sleep(0.03)
        continue

    # 画面水平翻转，更符合镜面操作直觉
    frame = cv2.flip(frame, 1)
    h_h, h_w, _ = frame.shape
    
    # 转换色彩空间以供给 MediaPipe 推理
    rgb_frame = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    hand_results = hands.process(rgb_frame)
    
    # 转换为灰度图供 OpenCV 人脸检测
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    faces = face_cascade.detectMultiScale(gray, 1.3, 5)

    # 决策标志
    detected_gesture = None
    finger_count = -1

    # --- A. 优先处理手势识别 ---
    if hand_results.multi_hand_landmarks:
        for hand_landmarks in hand_results.multi_hand_landmarks:
            # 绘制骨骼关键点
            mp_draw.draw_landmarks(frame, hand_landmarks, mp_hands.HAND_CONNECTIONS)
            
            # 获取 21 个骨骼点坐标
            landmarks = hand_landmarks.landmark
            
            # 简易手指张开计数算法 (食指、中指、无名指、小指)
            # 通过指尖(Tip)与第二指关节(Pip)的 Y 坐标对比来判定手指是否伸直
            fingers = []
            
            # 1. 大拇指判定 (根据 X 坐标相对位置判断，假设为右手/手心朝前)
            if landmarks[4].x < landmarks[3].x:
                fingers.append(1)
            else:
                fingers.append(0)
                
            # 2. 其他四个手指判定
            tip_ids = [8, 12, 16, 20]
            pip_ids = [6, 10, 14, 18]
            for i in range(4):
                if landmarks[tip_ids[i]].y < landmarks[pip_ids[i]].y:
                    fingers.append(1)
                else:
                    fingers.append(0)
            
            finger_count = fingers.count(1)
            cv2.putText(frame, f"Fingers: {finger_count}", (20, 80), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 255), 2)

    # --- B. 执行决策状态机映射 ---
    if finger_count != -1:
        # 手势具备最高优先级，当识别到手势时执行手势控制
        if finger_count == 0:
            # 握拳：关闭灯阵
            cv2.putText(frame, "Action: OFF", (20, 120), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 0, 255), 2)
            controller.send_mode(0)
        elif finger_count == 1:
            # 伸出 1 根手指：彩虹霓虹
            cv2.putText(frame, "Action: Neon Effect", (20, 120), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 0), 2)
            controller.send_mode(4)
        elif finger_count == 2:
            # 伸出 2 根手指 (剪刀手)：呼吸极光
            cv2.putText(frame, "Action: Aurora Effect", (20, 120), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 0), 2)
            controller.send_mode(5)
        elif finger_count == 3:
            # 伸出 3 根手指：流星划过
            cv2.putText(frame, "Action: Meteor Effect", (20, 120), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 255, 0), 2)
            controller.send_mode(6)
        elif finger_count == 4:
            # 伸出 4 根手指：启动贪吃蛇小游戏
            cv2.putText(frame, "Action: Launch Snake", (20, 120), cv2.FONT_HERSHEY_SIMPLEX, 1, (255, 0, 255), 2)
            controller.send_mode(19)
        elif finger_count == 5:
            # 伸出 5 根手指 (张开手掌)：启动俄罗斯方块
            cv2.putText(frame, "Action: Launch Tetris", (20, 120), cv2.FONT_HERSHEY_SIMPLEX, 1, (255, 0, 255), 2)
            controller.send_mode(21)
            
    elif len(faces) > 0:
        # 无手势，但检测到人脸时：进入人脸识别联动
        # 在画面中绘制框标识
        for (x, y, w, h) in faces:
            cv2.rectangle(frame, (x, y), (x+w, y+h), (255, 255, 0), 2)
            
            # 此处进行简易模拟：当人脸在画面中央且面积较大时发送“笑脸”
            face_area = w * h
            if face_area > 30000:
                cv2.putText(frame, "Emotion: HAPPY", (x, y-10), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (0, 255, 0), 2)
                controller.send_mode(11)  # 笑脸模式
            else:
                cv2.putText(frame, "Emotion: SERIOUS", (x, y-10), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 0), 2)
                controller.send_mode(13)  # 平静表情
    else:
        # 画面无任何人脸与手势：下发“中心波纹”作为呼吸待机态
        cv2.putText(frame, "Status: Idle (Wave Mode)", (20, 40), cv2.FONT_HERSHEY_SIMPLEX, 0.8, (255, 255, 255), 2)
        controller.send_mode(10)

    # 实时窗口预览
    cv2.imshow("Jetson Nano AI Controller", frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

# 释放所有物理资源
cap.release()
cv2.destroyAllWindows()
ser.close()
print("[OK] 程序安全退出。")
```

## 🎮 第六步：体感游戏控制的进一步设计方案 (致 Jetson 端组员)

当前版本的 ESP32-WROOM-32 固件在接收串口命令时，为了防止电磁空间辐射的零碎噪点干扰导致异常切模，**开启了纯数字校验**（即除了以 `f,` 开头的音律流之外，其余的非纯数字字符均会被拦截并不予理睬）。

如果你想通过**手势控制贪吃蛇的方向**或**俄罗斯方块的方块旋转/下落**，可以采用以下两个开发演进路线：

### 路线 A：固件端扩展物理串口解析（推荐，速度最快，延迟最小）
与负责 ESP32 端的同学协作，修改 `LightSystem3.ino` 中的 `processSerial2Command()`。允许直接解析方向控制字符：
- 接收到字符 **`'W'`** ➔ 触发游戏向上走 / 方块旋转
- 接收到字符 **`'S'`** ➔ 触发游戏向下走 / 方块加速下沉
- 接收到字符 **`'A'`** ➔ 触发游戏向左移
- 接收到字符 **`'D'`** ➔ 触发游戏向右移

在 ESP32 代码中的解析示意如下：
```cpp
// 在 processSerial2Command() 中拦截非数字前加入：
if (serial2Buffer == "W" || serial2Buffer == "S" || serial2Buffer == "A" || serial2Buffer == "D") {
    // 模拟 Blinker 手机按键的回调，改变 sDx/sDy/pX/pY 变量
    if (serial2Buffer == "W") buttonW_callback("");
    if (serial2Buffer == "S") buttonS_callback("");
    if (serial2Buffer == "A") buttonA_callback("");
    if (serial2Buffer == "D") buttonD_callback("");
}
```
此时你的 Jetson Python 代码只需要在贪吃蛇模式运行中，通过手势判定运动趋势（如手向左挥动），串口写入 `W`、`S`、`A` 或 `D` 即可。

### 路线 B：Blinker 网络控制流（无需改动固件）
利用 Blinker 的第三方控制 API。你的 Jetson Nano 脚本可以在本地识别出方向手势后，直接通过 Python 发送 HTTP 请求或 MQTT 消息到点灯科技云服务器，模拟 App 点击了控制面板上的方向按钮。
- *劣势：网络通信依赖 WiFi，相较于物理杜邦线，会有 100~300ms 左右的延迟，体感操控感可能有些许迟滞。*

## 🔍 第七步：现场联合调试与排错规程

1.  **若串口抛出 `Permission Denied` 错误**
    *   原因：系统未成功应用 `usermod` 组权限，或被其他进程死锁。
    *   解决：执行 `sudo chmod 666 /dev/ttyTHS1` 强行赋予全部用户临时读写权限。
2.  **若接线后 ESP32 完全不接收指令**
    *   原因：TX/RX 引脚插错，或者未共地。
    *   检测：确认 Nano 的 Pin 8 ➔ ESP32 GPIO 25，Nano 的 Pin 10 ➔ ESP32 GPIO 26。并且将两块板卡的任意 GND 物理引脚互连。
3.  **串口硬件回环自检（最有效排查方法）**
    *   方法：从 ESP32 上拔掉 Nano 的 Pin 8 和 Pin 10。用一根杜邦线将 Nano 自己的 **Pin 8 与 Pin 10 直接短路（物理环回）**。
    *   运行简单的 python 串口收发脚本。如果写入的数据能够原样被自己读取出来，说明 Jetson Nano 的系统配置、nvgetty 屏蔽以及硬件串口芯片均完全无故障。问题必定在 ESP32 侧或连线上。
4.  **Jetson Nano 硬件算力优化**
    *   如果你发现 MediaPipe 帧率过低（画面卡顿），请在 Jetson Nano 终端解锁最大功率模式并锁定主频：
        ```bash
        sudo nvpmodel -m 0          # 开启最大功耗模式 (MAXN)
        sudo jetson_clocks          # 强制锁定 CPU/GPU 时钟频率在最高点
        ```
