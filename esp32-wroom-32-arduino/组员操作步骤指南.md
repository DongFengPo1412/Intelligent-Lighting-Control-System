# Jetson Nano & ESP32 串口通信与调试操作指南

本指南用于指导组员如何使用杜邦线建立 Jetson Nano 与 ESP32-WROOM-32 之间的异步串口通信，并完成调试与脚本运行。

---

## 🛠️ 第一步：物理杜邦线接线指南

由于 ESP32 目前使用的是 **USB 连电脑供电**，且其 **`VIN` 引脚已经接入灯板的 `V+` 给灯板供电**。为了防止双路 5V 电源发生电流倒灌与冲突，**千万不要连接 Jetson Nano 的 5V 供电线到 ESP32**。

你只需要用杜邦线连接以下 **3 根线**：

| Jetson Nano J41 40-Pin 排针引脚 | ➔ | ESP32-WROOM-32 对应引脚 | 引脚功能与说明 |
| :--- | :---: | :--- | :--- |
| **Pin 8 (UART1_TXD)** | ➔ | **GPIO 25** (作为 RX2) | Jetson 发送 ➔ ESP32 接收 |
| **Pin 10 (UART1_RXD)**| ➔ | **GPIO 26** (作为 TX2) | ESP32 发送 ➔ Jetson 接收 |
| **Pin 9 (GND)** | ➔ | **GND** | **共用接地参考线** (必须连接，否则电平漂移) |

*注：GPIO 25 和 26 在 ESP32 固件中已通过引脚矩阵成功映射为 `Serial2` 的物理收发脚，运行稳定。*

---

## ⚙️ 第二步：Jetson Nano 串口服务配置 (重要)

Jetson Nano 的系统默认在 `/dev/ttyTHS1` 端口上启用了 `nvgetty` 串口终端服务。如果不关闭它，系统后台会与 Python 脚本互相争抢串口，导致数据丢失、乱码或失效。

请在 **Jetson Nano 终端** 执行以下命令关闭该服务：

```bash
# 1. 停止 nvgetty 服务
sudo systemctl stop nvgetty

# 2. 禁用 nvgetty 服务自启
sudo systemctl disable nvgetty

# 3. 触发 udev 规则并建议重启 Jetson Nano 使其彻底生效
sudo udevadm trigger
sudo reboot
```

---

## 🧪 第三步：极简终端测试 (点亮红灯)

在运行 AI 脚本前，先用最简单的终端命令验证物理接线与串口通信是否畅通：

1. **Jetson 终端开启串口读写权限**：
   ```bash
   sudo chmod 666 /dev/ttyTHS1
   ```

2. **发送点亮红灯测试信号**（ESP32 已集成 50ms 超时断帧，无换行符也能完美解析）：
   ```bash
   echo -n "1" > /dev/ttyTHS1
   ```
   或者发送标准带换行符的信号：
   ```bash
   echo "1" > /dev/ttyTHS1
   ```

3. **预期效果**：
   * 灯光板会瞬间**点亮全屏红色**。
   * 如果 ESP32 连着电脑，用 Arduino 串口监视器（波特率 115200）能看到打印日志：`>>> [串口2]: 收到控制信号: 1`。

## 🚀 第四步：运行 AI 识别脚本进行测试

确认极简终端测试通过后，即可在 Jetson Nano 上的 `py36` 文件夹内运行人脸/手势识别脚本（均已完成通信适配与 Bug 修复）：

### 1. 手势控制 1（手指个数控制 0~5 号效果）
```bash
python3 run_gesture.py
```
* **控制方式**：对着摄像头伸出不同数量的手指（0 到 5 个），灯光板会实时变幻对应的灯光效果。

### 2. 手势控制 2（特定手势控制特定效果）
```bash
python3 run_gesture2.py
```
* **控制方式**：
  * 做出“摊手”（所有手指伸开）➔ 发送 `5`
  * 做出“胜利✌️”手势 ➔ 发送 `6`
  * 做出“握拳”手势 ➔ 发送 `7`

### 3. 人脸识别控制
```bash
python3 face_system_cn.py
```
* **控制方式**：
  * 检测并识别到 `ylk` ➔ 发送 `1`（红灯）
  * 检测并识别到 `zzc` ➔ 发送 `2`（绿灯）
  * 人脸移开画面时自动重置发送状态。
  * **热键操作**：在视频窗口按 `S` 录入人脸；按 `T` 重新训练；按 `Q` 退出。
